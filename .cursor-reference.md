### Common RLS Policy Pattern
```sql
-- Select own data
CREATE POLICY "name" ON table_name 
  FOR SELECT USING (auth.uid() = user_id);

-- Insert with FK check
CREATE POLICY "name" ON table_name 
  FOR INSERT WITH CHECK (
    EXISTS (SELECT 1 FROM parent_table 
            WHERE parent_table.id = table_name.foreign_id 
            AND parent_table.user_id = auth.uid())
  );
```

## Code Patterns Quick Ref

### Server Action (Do This)
```typescript
'use server'

export async function action() {
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()
  
  if (!user) throw new Error('Not authenticated')
  // Always filter by user.id for RLS
}
```

### Client Component (Do This)
```typescript
'use client'

import { createClient } from '@/lib/supabase/client'

export default function Component() {
  const [state, setState] = useState(null)
  
  useEffect(() => {
    const supabase = createClient()
    // Client operations
  }, [])
}
```

### Realtime Subscription (Pattern)
```typescript
const subscription = supabase
  .channel(`room:${id}`)
  .on('postgres_changes', { 
    event: '*', 
    schema: 'public', 
    table: 'table_name',
    filter: `column=eq.${userId}`
  }, (payload) => {
    console.log('Change:', payload)
  })
  .subscribe()

return () => subscription.unsubscribe()
```

## Testing a Feature

1. **Create Quiz**
   - Go to `/dashboard/create`
   - Fill form and create
   - Should redirect to quiz editor

2. **Add Questions**
   - In quiz editor, click "Add Question"
   - Add MCQ, set answers
   - Should appear in list

3. **Launch Session**
   - Click "Launch" on quiz card
   - Get PIN code
   - Should show session details

4. **Join as Student**
   - Visit `/join`
   - Enter PIN code
   - Enter nickname
   - Should load quiz player

5. **Play Quiz**
   - Select answer
   - Submit
   - Move to next question
   - See results

## Common Issues & Fixes

| Issue | Cause | Fix |
|-------|-------|-----|
| "Not authenticated" error | User not logged in | Check auth/login page |
| RLS denies insert | Missing user_id | Ensure auth.uid() = user_id |
| Questions not loading | Quiz ID mismatch | Verify quiz.id in URL |
| Realtime not working | No subscriptions set | Add .subscribe() call |
| Styles not applying | Design token not set | Check globals.css |

## Performance Tips

- Use `revalidateTag()` after mutations
- Lazy load heavy components: `const X = dynamic(() => import('./X'))`
- Memoize callbacks: `useCallback((x) => {}, [])`
- Use `suspense` for async components

## Debugging Checklist

- [ ] Check browser console for errors
- [ ] Verify Supabase connection: `supabase.auth.getSession()`
- [ ] Test RLS by checking auth.uid() filter
- [ ] Use Chrome DevTools Network tab for API calls
- [ ] Check Supabase Studio for data integrity
- [ ] Verify environment variables are loaded: `console.log(process.env.NEXT_PUBLIC_SUPABASE_URL)`
